# GLM 5.2 — Zhipu AI / Z.ai GLM Family

Research date: 2026-06-29. GLM-5.2 shipped to paying coding customers on ~June 13, 2026 and weights went public mid-June 2026.

## What it is

- GLM-5.2 is Z.ai's (formerly Zhipu AI) flagship open-weights LLM, positioned as the strongest *openly available* model for long-horizon autonomous coding and agentic engineering ("From Vibe Coding to Agentic Engineering").
- Mixture-of-Experts architecture: **~744–753B total parameters, ~40B active per token**. Repo headline figure is 744B total / 40B active; press coverage rounds to 753B.
- Precision options: **BF16 and FP8**; community quants (GGUF for llama.cpp, NVFP4/REAP ~504B slimmed variants, Unsloth) appear within days of release.
- Lineage: GLM-4.6 (Sept 2025, 355B MoE, 200K context, MIT) → GLM-5 / GLM-5.1 ("Towards Long-Horizon Tasks") → GLM-5.2.

## Context window & I/O

- **1,000,000-token context window**, marketed as "stably sustained" for long-horizon work (not just a nominal max).
- **Up to 131,072 (128K) output tokens.**
- Efficiency innovation **"IndexShare"**: reuses sparse-attention top-k indices across groups of layers, cutting per-token compute at 1M context by **~2.9×**. This is what makes 1M context economically practical despite the large parameter count.
- Upgraded **Multi-Token Prediction (MTP)** layer for speculative decoding — boosts accepted token length up to ~20% during inference.

## Reasoning / thinking model

- Reuses the **DeepSeek-V4-style `reasoning_effort` mechanism**. Two effort levels: **`high`** and **`max`**.
- The chat template resolves effort to **`max` by default**; `high` is opt-in (lower latency/cost).
- **Thinking is enabled by default** — responses return both reasoning and content tokens. Supports **interleaved thinking** (model thinks between tool calls and after tool results).
- Disable thinking via `chat_template_kwargs {"enable_thinking": false}`; `reasoning_effort` has no effect when thinking is off.

## Tool / function-calling format

- **Standard OpenAI-compatible tool schema**: tools passed via `tools` array, each `{"type":"function","function":{name, description, parameters:{properties, required}}}`.
- Tool calling is **interleaved with reasoning** — model reasons between each tool step and after receiving results.
- **Streaming caveat**: tool-call arguments are emitted incrementally; clients must **concatenate argument fragments across deltas** to rebuild the full call. (Important for a harness to implement correctly.)
- **Structured output**: JSON Schema via `response_format` constrains output to a fixed shape.
- Hosted Z.ai API base URL: `https://api.z.ai/api/paas/v4/`, model id `glm-5.2`. OpenAI-compatible: chat, thinking, streaming, tool calling, token tracking.

## Where it beats / ties Claude

- **Front-end / web UI generation** and **design** — #1 on Design Arena, ahead of Claude Fable 5; visually polished front-end pages a noted strength (continuing GLM-4.6's reputation).
- **SWE-bench Pro** coding — beats GPT-5.5; competitive with Opus on real coding.
- **Cost-per-task economics** — ~$2.40/task vs Claude Fable 5's ~$31 (~13× cheaper); ~1/6th the cost of GPT-5.5 on the same long-horizon coding work.

## "Center of distribution" strengths

- Community consensus: GLM-5.2 excels at **front-end dev, web apps, design, presentation/deck creation, and "routine synthesis"** — high-frequency, well-trodden tasks. Multiple practitioners report it clears their "daily driver" bar.
- **Where premium proprietary models still win**: long-horizon reasoning and *novel* problem-solving. Note even frontier models satisfy *all* rubric criteria on only ~3% of multi-week knowledge-work tasks — the hard tail is unsolved across the board. The economic argument: pay premium only for the tail; serve the center of the distribution with GLM.
- Terminal-Bench / agentic-coordination gaps vs Opus 4.8 and GPT-5.5 remain (81 vs 84–85), i.e. very long autonomous terminal sessions still favor the closed flagships.

## Pricing — cloud vs self-hosted

- **Z.ai cloud subscription**: enterprise/coding tiers from **~$12.60/month**.
- **Serverless API (e.g. Fireworks)**: GLM-5.2 ≈ **$1.40 / $0.14 (cached) / $4.40** per 1M tokens (input/cached-input/output), Standard tier. Roughly **~5× cheaper and ~4.5× faster than Claude Opus 4.8** per Baseten.
- **Self-hosted**: open weights are free; cost is GPU. A 744B/40B-active FP8 model needs multi-GPU (e.g. 8×H100/H200 class). On-demand GPU reference (Fireworks): H100 80GB / H200 141GB ≈ $7.00/hr, B200 ≈ $10/hr, B300 ≈ $12/hr. Self-hosting wins at high sustained volume; serverless wins for spiky/low volume.

## Open weights & license

- **Weights public on Hugging Face and ModelScope.**
- **License**: the GLM-5 repo states **Apache-2.0**; some launch press described it as **MIT**. Both are permissive (commercial use, self-host, fine-tune, no regional lock). **Confirm the exact LICENSE file in `zai-org/GLM-5.2` before relying on it** — the repo (Apache-2.0) is the authoritative source.
- Serving frameworks supported day-1: **SGLang, vLLM, HF Transformers, KTransformers, Unsloth**, plus Ascend NPU. vLLM and SGLang have published recipes for `zai-org/GLM-5.2`.

## Bundled Codex-clone harness

- GLM-5 ships oriented around agentic coding harnesses. There is an open-source **`GLM-codex`** ("lightweight coding agent that runs in your terminal", GitHub `2404589803/glm-codex`) — a Codex-style terminal agent wired for GLM models.
- GLM-5.2 is explicitly designed to **drop into existing harnesses** via its OpenAI-compatible endpoint: Claude Code, OpenAI Codex CLI, Cline, Cursor, Roo Code, Kilo Code, OpenCode, LangChain Deep Agents CLI, Pi.
  - Codex example: profile at `~/.codex/*.config.toml` with `model = "zai-org/GLM-5.2"`, `model_reasoning_effort`, OpenAI-compatible `base_url` (e.g. provider's `/v1`).
  - Claude Code: point `ANTHROPIC_BASE_URL`-style env at the provider and set the GLM model across tiers (Anthropic-compatible shims exist on providers).

## Key takeaways for glamfire

- **Target model**: GLM-5.2 = 744B/40B-active MoE, 1M context, 128K output, thinking-by-default with `reasoning_effort` (`high`/`max`). glamfire should expose `reasoning_effort` and an `enable_thinking` toggle as first-class config.
- **Tool calls are OpenAI-schema + interleaved-reasoning + fragmented streaming** — glamfire's tool-call parser MUST accumulate argument deltas, and the loop MUST tolerate reasoning tokens interleaved between tool calls (don't assume one-shot tool blocks).
- **Default to `high`, escalate to `max`** for hard tasks to control latency/cost, since `max` is the template default.
- **Play to the center of distribution**: route front-end/web/design/deck/routine-synthesis work to GLM-5.2 confidently; reserve a premium fallback for the long-horizon/novel tail where Opus 4.8 / GPT-5.5 still lead.
- **Economics are the whole pitch**: ~5× cheaper, ~4.5× faster than Opus; build cost-per-task telemetry to prove it.
- **License**: verify Apache-2.0 vs MIT from the repo LICENSE; both permit self-hosting and commercial use, removing API lock-in.
